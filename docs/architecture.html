1Two-Model Design
Encoder multilingual-e5-small
BERT-family, encoder-only. Maps text → 384-dim dense vector for semantic search. Cannot generate.
Decoder gemma4:e4b
Gemma-family causal LLM (e4b), 4-bit GGUF. Autoregressive generation & explanation. Cannot produce fixed-size embeddings.
  • Right tool per task — retrieval needs an embedding model, generation needs a causal LM

3Prompt Engineering
  • Domain-routed system prompts — 4 variants by question subtype: math · language · civics (TWK) · situational (TKP)

  • "Leave the math open" — LLM writes the expression ending in = and the symbolic engine fills the result, so the model never does arithmetic

5Neuro-Symbolic Verification — anti-hallucination layer
  • The LLM proposes; deterministic Python disposes. Generated answers are never trusted on arithmetic — a symbolic parser re-computes from the question text and reconciles.
CONFIRMcomputed answer matches an option → lock correct_label to the verified one
PATCHanswer computed but absent from options → LLM hallucinated → overwrite that option with ground truth
CORRECTno math detectable → 2nd LLM "validator" pass + Python re-reads the explanation as final arbiter

Constrained decoding
  • All LLM calls use JSON-grammar-constrained decoding (response_format=json_object) — output is guaranteed parseable, no regex scraping
7Local LLM Inference
  • gemma4:e4b via llama.cpp bindings — model runs in-process; no inference server, no HTTP, no port
  • 4-bit quantization (GGUF) — shrinks weights ~4× so the model fits in unified memory
  • Metal GPU offload — n_gpu_layers=-1 runs all layers on Apple Silicon
8Streaming & Cancellation
  • Token-level streaming — the generation loop checks a cancel flag between tokens
  • Cooperative abort — if the user navigates away, inference stops mid-generation in <100 ms instead of burning GPU on a discarded result
9Key ML Decisions
Choice Why
Local quantized LLM Data privacy, no API cost, full cancellation control vs. hosted APIs
Encoder for RAG E5 gives fixed-size vectors; a decoder can't — different model classes per job
Flat cosine, no FAISS Small corpus fits in RAM; exact search beats approximate-NN infra here
Symbolic verifier LLMs hallucinate arithmetic — re-compute & patch, never trust the model on numbers
Constrained JSON decode Guarantees parseable structured output; removes a class of failures
Scene-graph, not pixels Text LLM generates visual specs, deterministically rendered — no diffusion model
